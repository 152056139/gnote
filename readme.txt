1、按照从需求分析到测试5个文档模版格式及要求对所做的记事本软件项目进行描述。

2、软件功能不一定追求功能“大”、“全”，但要求做的“精”。要通过独立完成一个哪怕是“简单的小项目”，真正体会项目过程。

3、如果一个项目由多人共同完成（最多3人1组），则要求文档也按照各自的任务分配独立完成自己的部分（可每个人负责完成其中的一个或几个文档）。

4、模版中蓝色字体部分是说明本部分内容的写作要求，在完成文档时自行删除，但其版面格式不能随意改变。

5、请项目组长负责组织项目组成员对相关文档进行评审，只有评审通过后的文档才能提交。如果提交的文档不符合规范或存在明显的错误，文档将会被退回重写并扣除项目组一定的分数。

6、每个班由学习委员牵头，项目组长为成员，组成2-3个评审组，交叉对每个组的相关文档进行评审，只有评审通过后的文档才能正式打印提交。如果提交的文档不符合规范或存在明显的错误，文档将会被退回重写并扣除评审组成员和文档撰写者一定的分数。

7、可以通过文档的不同版本体现版本迭代过程。

8、文档的提交和评审时间节点通过平台发布。
